# Contributing to Friendly Friends App

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/friendly-friends-app.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -m "Add: description of changes"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Code Style

### Python (Backend)
- Follow PEP 8 style guide
- Use 4 spaces for indentation
- Maximum line length: 100 characters
- Use descriptive variable and function names

### JavaScript/React (Frontend)
- Use ES6+ syntax
- Use functional components with hooks
- Follow React best practices
- Use meaningful component and variable names
- Use camelCase for variables and functions
- Use PascalCase for components

## Commit Messages

Use clear, descriptive commit messages:
- `Add: new feature description`
- `Fix: bug description`
- `Update: change description`
- `Remove: removed feature description`
- `Refactor: refactoring description`

## Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Backend: Run `pytest` in the backend directory
- Frontend: Run `npm test` in the frontend directory

## Pull Request Process

1. Update README.md if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update documentation
5. Request review from maintainers

## Questions?

Open an issue for questions or discussions about the project.
